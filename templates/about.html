About the Project

This project uses a machine learning model to classify images of weeds. The model was trained using a custom dataset of various weed species to provide accurate and reliable predictions.

Weed Detection Model Training - Steps Overview

Google Drive Integration

Mounted Google Drive to access and store datasets and trained models.

Data Loading & Preparation

  • Loaded weed species dataset from Google Drive.
  • Automatically identified classes (weed categories).
  • Split data into training (80%) and validation (20%) sets.

Data Augmentation

Applied transformations such as flipping, rotation, zooming, and translation. This improved dataset diversity and helped prevent overfitting.

Transfer Learning with MobileNetV2

Used MobileNetV2 pre-trained on ImageNet as the base model. It was fine-tuned with custom layers for binary classification of weed species.

Model Compilation

  • Optimizer: Adam (learning rate 0.0001).
  • Loss Function: Binary Crossentropy (for two-class classification).
  • Metrics: Accuracy.

Model Training

The model was trained for 15 epochs on the augmented dataset and its performance was evaluated using validation accuracy and loss.

Model Saving

The trained model was saved to Google Drive for future use and deployment.

Training Visualization

Plotted accuracy and loss graphs for both training and validation sets to monitor performance.

Prediction Example

Tested the model on a sample image from the dataset, and prediction results with a confidence percentage were displayed.
